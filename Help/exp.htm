Expressions

The ZlibStream object has 4 expressions.

Returned string
Percentage
Last output file
Buffer size

Returned string:
ReturnedString$("ZlibStream")
This expression gets the description of the last error, or the description of the last successful compression/decompression action. Use this when either of the conditions are set to TRUE.
Parameters: None.

Percentage:
Percentage("ZlibStream")
This expression gets the size difference as a percentage (%) between the input file and the output file of the last successful compression/decompression action, accurate to two decimal places. For example, it can return 3.05, 114.30, etc.
Parameters: None.

Last output file:
LastOutputFile$("ZlibStream")
This expression gets the last output file path of the last successful compression/decompression action.
Parameters: None.

Get buffer size:
BufferSize("ZlibStream")
This expression gets the default buffer size, which is a number from 1 to 16384; by default it is 8192. Each compression/decompression loop will read that number of bytes from the input file - although it will write less or more bytes to the output (depending on whether it is compressing or decompressing). This is approximately the amount of new memory used by ZlibStream when you start a new compression/decompression action. You can change it with the Set buffer size action. If the buffer size is larger than the size of the input file, the buffer is automatically made smaller for that input file only. Other actions are not affected (and this number is not changed). You can see the actual buffer used in the last completed compression/decompression action as part of the ReturnedString$("ZlibStream") expression.
Parameters: None.
