Actions

The ZlibStream object has 3 actions.

Compress a file
Decompress a file
Set buffer size (bytes)

Compress a file:
This action initiates a compression of a file. It opens the file specified in the first parameter, and starts a compression loop where the input file is read, compressed, and copied into the output file. This is done at a rate of 1 to 16384 bytes per loop - by default it is 8192. You can change the rate by changing the buffer size using the Set buffer size action. After the compression finishes, you can use the LastOutputFile$("ZlibStream") expression to get the last completed compression output file.
The third parameter is simple: if the output file exists, do you want ZlibStream to overwrite it (erase the old contents) or add the new compression to the end(appending)? Use 0 to overwrite, 1 to append.
Parameters: Input file (string, filepath), Output file (string, filepath), Use append (number between 0 and 1).

Decompress a file:
This action initiates a decompression of a file. It opens the file specified in the first parameter, and starts a decompression loop where the input file is read, decompressed, and copied into the output file. This is done at a rate of 1 to 16384 bytes per loop - by default it is 8192. You can change the rate by changing the buffer size using the Set buffer size action. After the decompression finishes, you can use the LastOutputFile$("ZlibStream") expression to get the last completed decompression output file.
The third parameter is simple: if the output file exists, do you want ZlibStream to overwrite it (erase the old contents) or add the new decompression to the end(appending)? Use 0 to overwrite, 1 to append.
Parameters: Input file (string, filepath), Output file (string, filepath), Use append (number between 0 and 1).

Set buffer size (bytes):
This action changes the rate that bytes are processed by ZlibStream. Use a lower number to compress/decompress slower and use less memory, and a higher number to compress/decompress faster and use more memory. The limit is 1 to 16384 bytes per loop. Any new compressions/decompressions you start then use that buffer size. This number should also be the amount of new memory taken up by ZlibStream with each compression/decompression action.
Parameters: Buffer size (number between 1 and 16384).
